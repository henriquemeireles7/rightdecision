# Product Vision

> Last verified: 2026-04-09

## The Decision OS

The Right Decision is a Decision Operating System. Every product is a surface on the OS:

| OS Layer | What It Is | Phase |
|----------|-----------|-------|
| **Kernel** | The methodology ("deciding is the primitive") | 1 |
| **User Layer** | Course, Skills, Decision Graph (personal history) | 1 |
| **Social Layer** | Decision Network, Wins Board, Decision Twins, Decision Birthday | 1-2 |
| **Developer Layer** | Decision Protocol (open standard), API, Marketplace | 3 |
| **Intelligence Layer** | Own AI model, Collective Decision Intelligence, Research Institute | 3-4 |
| **Financial Layer** | Stripe Connect, Banking-as-a-Service | 2-4+ |
| **Runtime Layer** | On-device model, custom silicon | 4-5 |

## The Data Flywheel

```
User decides → Decision Graph captures it → Network patterns emerge →
Model improves → Better decisions → More users → (repeat)
```

Every project accelerates this flywheel. Every design decision asks: "Does this generate structured decision data?"

## Two Products, One Platform

- **Life Decisions** ($197/year) — Personal decision methodology + AI exercises for women 30-50
- **Business Decisions** ($1,997/year) — White-label platform for non-tech entrepreneurs to run their own Decision course

## 5 Tests for Every Product Decision

1. **Flywheel?** Does it generate structured decision data?
2. **Load-bearing?** Will this serve us in Phase 2+?
3. **Schema?** Does the Decision Graph support it?
4. **Anti-self-help?** Is this about action, not introspection?
5. **Dogfood?** Will Henry + Indy use this themselves?

## Non-Negotiable End State

Own model + own hardware. Full stack ownership. The infobusiness is GTM for a vertical AI company.
