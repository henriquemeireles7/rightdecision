## On growth categories

"Conversion: l1 - manually creating tests, L2 using skills to do A/B testing to improve conversion rates, L3 automated scheduled new tests against the winner, L4 self-learning better hypothesis with each test, L5 self-evolving tuning its own parameters to improve the self-learning. Same thing in expansion: manual, using skills, automated scheduling, self-learning, self-evolving. That is the pattern EVERYWHERE."

## On universal maturity pattern

"L2 is about assisted with skills, L3 is about automate by scheduling, L4 is about moving the metric not only automating but actually the automation makes the system improve itself, so L5 should be about self-evolving the system meta-evolves, tuning itself"

"L2 - lots of upgrades to the harness itself, like we are doing right now. L3 - it means we start to schedule things then the system starts to run itself and I start to only check the end-work and maybe update the harness based on the end work. L4 - it means we will have to focus hard on analytics and A/B testing on L4, this is where we not only automate but somehow we have a self-learning loop that shows what happened and we start to generate better hypothesis over time. L5 - is the hardest to think about it, its about the AI system itself wiring its own harness, making its own improvements, this is beyond self-learning this is something like recursive self-learning"
