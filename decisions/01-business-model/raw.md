# RAW: Business Model v3 — Two Products, One Brand

Captured: 2026-04-06
Source: Conversation between Henry and Claude about expanding from one B2C product to two products.

---

## The pivot moment

After completing docs 1-9 (strategy foundation), Henry realized the "internal software" (content pipeline, distribution automation) IS the B2B product. Not a separate thing built later — the same thing, productized for non-tech entrepreneurs.

"the idea is that we would have a course+software for the frontend (life decisions - teaching mainly woman to make better decisions at life) and a course+software for the backend (company decisions - teaching how to create and run a business the same way me and indy are running this business, basically use our platform to get your course ready and our 'internal software' to do your distribution)"

## Two products crystallized

"So it is kinda that our 'decisions' folder would be converted into a course for b2b - we could even share our docs with our clients - because the idea is that they will do the same thing."

Life Decisions = the original product. Course + AI for personal decisions.
Business Decisions = NEW. Course + automation platform for building AI-native infobusinesses.

"they are all non-tech people, so my challenge is to create a non-tech platform they can make money"

## The naming

"Now it means that the three-track structure is life decisions and business decisions (I feel like these are the names for now)."

## Pricing

"One product, 2 tiers. Life Decisions $197/y - Business Decisions $1997/y (this is 10x expensive, because the user will also needs to pay for claude so it is a high ticket product)"

## Three personas

Persona 1: Woman 30-50, stuck at a higher level (Life Decisions) — original ICP
Persona 2: Man 25-40 who overthinks career/business moves, consumes business content but never starts (Life Decisions male angle)
Persona 3: Non-tech entrepreneur 28-45 who has an idea or early business but drowns in planning instead of executing (Business Decisions)

## Skills as the product

"Yes. Its one skill per major exercise. The idea is that we should learn from gstack how to make skills that will help the user to understand themselves."

Each methodology step has its own Claude skill. The exercise class teaches/shows the user what the skill does. Skills use the same meta → input → document pattern as our d-meta → d-input → d-plan.

"so in the end we will kind of use the 'same methodology' in both life and business"

## The AI course pivot

"AI workflow should be a standalone bonus course, not part of the main 3-act structure. Keep the core course pure methodology."

Two bonus courses:
- Life Decisions bonus: "Claude Cowork for Personal Decisions"
- Business Decisions bonus: "Vibe Coding for Non-Tech Creators/Founders"

"but the main course only have like 1 course in introduction helping to setup and explaining the AI part of the course"

## The agent-first vision

"I feel like we just need to create APIs and SKILLS to compose these APIs - so that the user can choose claude, claude cowork or claude code"

"I feel like we should build this agent-first so actualy instead of clicking into the platform we would have a claude skill that uses our APIs to do stuff"

"So maybe in the end we don't need neither desktop or web app. Everything happens in the file system."

## The storage constraint

"the constraint here is that the file system will get massive and it wont work anymore because we are giving it high-quality videos. So maybe we do need a storage, but IDK if that means we need a vps, a platform, a google drive, or how to connect this with claude"

## Life Decisions doesn't need API

"lifedecisions do not need API, as this is docs only, the API is only necessary for the business to do stuff like posting or measuring or improving stuff - so it means that the lifedecisions do not need an API, just a skill for each exercise"

## The free course funnel

"we could actualy focus the ENTIRE social media communication into this 'free course funnel' that the user will join our platform (so we get its email) and also he will have a great free course and he will see that he can actualy buy the entire thing the runs WITH claude"

"the 'free course' could be a the simple methodology version with maybe less steps but it works by itself"

"AND this 'free course' methodology is also what we will teach our clients in the business decisions - so we teach them exactly what we do. That is the whole flywheel."

## Docs as course content

"we could even share our docs with our clients - because the idea is that they will do the same thing"

"Because our docs ARE THE COMPANY itself. You see the value in giving away our documents for example? because clients would actually see an example of their company docs."

## The cross-sell

"There is a cross-sell in the end-user application. For example, both products needs a course."

## Henry and Indy doing the exercises

"Now thinking about this I even feel like maybe me and indy will have a 'life decisions' folder and actualy do all exercises so that we can show this to our users in our platform."

"we will do this later, we have done some planning ourselves outside the computer and I dont wanna mix things up yet"

## The content angle

"the main cause behind not making decisions is the complexity so thats why we use AI to help us have more clarity"

"business decisions to make money running an AI-native info business: vibe coding for non-tech creators/founders"

"we could for example giveaway one pdf with prompt and in this pdf we would teach users to use it AND show our full product"

## What the Business Decisions client gets

"they would only do 'decisions' and 'content' folder - and everything else would be abstracted for them"

"they would have a course+distribution software done for them and somehow they would use it without needing to code the platform themselves, THAT is the product"

## Architecture insight

"software (the plumbing is here, everyone can use it) / lifedecisions (the decisions+content is here so that the user would kind of run this business with only these folders) / businessdecisions (the same thing)"

"the same 'lifedecisions' folder architecture WE HAVE should work for our clients"

## Open questions (Henry's words)

1. "I have no idea how to do this yet" — how non-tech users run the platform
2. "IDK if that means we need a vps, a platform, a google drive" — storage for video files
3. "I was thinking that if my wife records the podcast, she would need to send to my computer" — practical file transfer
4. "are we building a saas or maybe a kind of desktop app to run a business?" — product form factor
5. "IDK how that would work" — agent-first platform interaction model
